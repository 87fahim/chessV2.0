Recommended product scope

Your chess app can be split into these major modes:

1. Play vs Computer
User chooses white/black
Difficulty levels
Move validation
Undo / redo
Resign / draw / restart
Save game history
Timers optional
2. Practice / Free Mode
Drag and drop any piece anywhere
Add/remove pieces from board
Clear board
Load standard starting position
Import/export FEN
This mode should ignore legal-move restrictions if desired
3. Find Next Move
User sets board position
App sends FEN to backend
Backend uses engine to suggest best move
Show:
best move
evaluation
principal variation
optional top 3 moves
4. Online Play
Real-time matchmaking
Invite friend
Game rooms
Reconnect if disconnected
Spectator mode later
Chat later
5. Guest vs Logged-in User
Guest:
quick play
local session only
maybe temporary saved game in browser
Logged in:
saved games
preferences
history
ratings
puzzles progress
multiplayer stats
Requirements you are likely missing
Functional requirements
Chess rules

You need to define whether your app supports all of these from day one:

legal move validation
check / checkmate
stalemate
castling
en passant
promotion
draw by repetition
draw by 50-move rule
insufficient material
move history
PGN export/import
FEN import/export

Without clearly defining this, the app becomes messy later.
